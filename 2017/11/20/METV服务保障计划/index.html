METV 服务保障计划

2017-11-20

已有成果

在公司网络条件下，表现已经超过应用市场上的直播应用，与讯码接近，与IPTV盒子还有较大差距。（感性认识）

进一步目标

在达成在公司网络条件下表现良好的前提下，我们进一步的目标应为在各地真实使用环境下，做到行业领先。在真实使用环境下领先可以使盒子具有实际上的竞争优势。
我们可以通过直接指标量化METV的表现，但领先的判断标准可能是与其他产品（包括IPTV盒子）感性对比。

当前目标

在重点地区（广东、山东、河南、上海），重点频道（央视、卫视、当地地方台）的能播概率、流畅程度、平均起播速度、清晰程度（当设置为“高清优先”时）达到较高水平。
未来发展路线应该一步步扩大重点地区的范围，最后再追求频道数量。

实现目标的阻碍和解决方向

目的 阻碍 解决方向
获取地区、运营商维度的数据 无法评判各地区实际使用情况 通过多维度的的数据，了解各地区、网络情况下的频道质量
提高各地区、运营商维度下的频道质量 对于新源或顺序调整，没有具有说服力的上线依据 在重点地区安排经销商/其他人员进行测试
提高各地区、运营商维度下的频道质量 对于新源或顺序调整，没有具有说服力的上线依据 A/B测试，根据数据表现上线
提高各地区、运营商维度下的频道质量 针对某个地区/用户，预设的源顺序很可能不够好 通过提供易用的工具和适时的提醒，让用户自行优化
提高各地区、运营商维度下的频道质量 针对某个地区/用户，预设的源顺序很可能不够好 建立客户端本地源顺序自动优化的机制
提高频道数量 没有稳定的获得地方台、轮播台的渠道，且很难进行保障 提供用户主动上传频道/源的方法

解决方向详情

通过多维度的的数据，了解各地区、网络情况下的频道质量

增加维度，以了解各地区、网络运营商下的直播质量。
明确指标定义，以多角度反映直播质量。

维度：

  1. 地区：粒度到省
  2. 运营商：主要运营商

指标：

  1. 频道数量
  2. 频道质量（通过源质量、源顺序、源数量保证。源质量决定频道质量上限，源顺序决定频道质量是否能达到上限、源数量决定容灾能力）
    1. 能播概率
      成功播放次数/(成功播放次数+播放失败次数+加载2秒后切台次数)
    2. 流畅程度
      每10分钟平均卡顿次数
      • 播放器知道卡顿，但由于码率太高导致跳帧播放器不知道
      • 区分流畅优先、默认、高清优先（原因是不同设置会影响表现，且不同设置侧重的点不同）
    3. 清晰程度
      开始播放时分辨率在某个尺寸以上的占比
      • 区分流畅优先、默认、高清优先（原因是不同设置会影响表现，且不同设置侧重的点不同）
    4. 平均起播速度
      从切换到该频道到频道开始播放的平均时间
      • 区分流畅优先、默认、高清优先（原因是不同设置会影响表现，且不同设置侧重的点不同）

在重点地区安排经销商/其他人员进行测试

每当有一批不确定质量的源上线，可以给经销商或其他人员一个用于测试该批源的APK以测试当地情况。
若有可能，应收集这部分APK上源的使用数据，以提供更可靠的判断依据。

A/B测试，根据数据表现上线

为了保证新上和修改后源的质量，可以进行部分上线，以对比测试。
部分上线的方法可以是渠道、UUID尾数等。理论上通过UUID尾数部分上线所得出的结果比较准确。
A/B测试机制的建立可为后面的产品优化提供工具。

通过提供易用的工具和适时的提醒，让用户自行优化

问题 原因 解决方法
频道卡顿 源质量不佳 切源、设置“频道加速”
频道卡顿 网络问题 检查网络连接、设置流畅优先
频道起播慢 网络问题 检查网络连接、设置流畅优先
频道清晰度低 源质量不佳 切源、设置高清优先、设置“频道加速”
频道清晰度低 分辨率问题（极少出现） 设置盒子分辨率

上面表格列出了用户可以且愿意主动优化的问题，对于这些问题，可以进行以下建议：

  1. 当出现多次卡顿时，提醒用户进行切源
  2. 当多个频道出现卡顿时，提醒用户进行智能切源设置（每次启动只提示一次；若已设置为流畅优先，不提示）
  3. 当清晰度低于一个极低数值时，提醒用户进行切源（每次启动只提示一次；若已设置为流畅优先，不提示）

（不提示用户进行“频道加速”设置，一般自动设置就是最佳设置，提醒用户反而让用户改错）

建立客户端本地源顺序自动优化的机制

根据以下优先级，进行源顺序自动优化。暂定规则如下，具体规则待完善：

  1. 智能切源设置
  2. 用户手动切源数据
    • 手动切换并持续播放1min的源，手动切源指数+1，手动切换走后，手动切源指数-1
    • 手动切源指数不小于0、不大于2
  3. 用户本地播放数据
    • 本地播放时满足优质源指标的，记录为本地优质源
    • 本地优质源分流畅和高清两种，指标不同
    • 每次源列表有更新时，本地优质源重置
  4. 地区+运营商筛选条件下的播放数据
    • 播放数据满足优质源指标的，记录为优质源
    • 优质源分流畅和高清两种，指标不同
    • 每10min评价一次，与数据监控同步
  5. 预设源顺序
    • 即默认源顺序

提供用户主动上传频道/源的方法

非当前目标。

除了提供更多频道外，可以帮助避险。
上传的频道/源立即出现在自定义频道列表中，若通过数据判断表现不错，可以加入频道/源列表。

服务保障计划和责任人

这里只列出预期的项目，具体计划增减和责任人待会后决定。

  1. 数据监控
  2. 日常检查频道质量
  3. 默认源顺序调整
  4. 问题反馈
